# Инструкция по запуску BoomBet Frontend

## Быстрый старт

1. **Установите зависимости:**

   ```bash
   cd boombet-frontend
   npm install
   ```

2. **Настройте переменные окружения:**

   ```bash
   cp .env.example .env
   ```

3. **Запустите проект:**

   ```bash
   npm run dev
   ```

4. **Откройте браузер:**
   Перейдите по адресу: http://localhost:3000

## Архитектурные особенности

### Структура состояния

Приложение использует Zustand для управления состоянием:

- **authStore** - аутентификация, пользователь, баланс
- **matchesStore** - матчи, коэффициенты, live обновления
- **betSlipStore** - купон ставок, выбранные события

### API интеграция

- Базовый URL: `http://localhost:8080/api`
- Автоматическое добавление JWT токенов
- Обработка ошибок аутентификации
- WebSocket для real-time обновлений на порту 8080

### Основные маршруты

- `/` - Главная страница
- `/live` - Live матчи
- `/prematch` - Предстоящие матчи
- `/login` - Вход в систему
- `/register` - Регистрация
- `/profile` - Профиль пользователя (защищенный)
- `/bets` - История ставок (защищенный)

### Дизайн система

- Темная тема по умолчанию
- Tailwind CSS для стилизации
- Responsive дизайн
- Кастомные компоненты для ставок

## Интеграция с Backend

Фронтенд ожидает следующие API endpoints от Java backend:

### Аутентификация

- `POST /api/auth/login` - Вход
- `POST /api/auth/register` - Регистрация
- `GET /api/auth/profile` - Профиль пользователя
- `POST /api/auth/logout` - Выход

### Матчи

- `GET /api/matches` - Список матчей с фильтрами
- `GET /api/matches/live` - Live матчи
- `GET /api/matches/top` - Топ матчи
- `GET /api/matches/{id}` - Конкретный матч

### Ставки

- `POST /api/bets/place` - Размещение ставки
- `GET /api/bets/history` - История ставок
- `GET /api/bets/active` - Активные ставки

### WebSocket события

- `odds_update` - Обновление коэффициентов
- `match_update` - Обновление матча (счет, статус)
- `balance_update` - Обновление баланса
